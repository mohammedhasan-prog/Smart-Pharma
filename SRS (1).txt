📄 SYSTEM REQUIREMENTS SPECIFICATION (SRS)
Smart Pharmacy Inventory Management System
1. Introduction
1.1 Purpose

The purpose of this system is to develop a web-based Smart Pharmacy Inventory Management System that automates medicine stock tracking, expiry monitoring, and restocking processes. It minimizes manual intervention by enabling automatic order generation and communication with wholesalers.

1.2 Scope

The system provides functionality for two types of users:

Seller (Pharmacist): Manages inventory, sells medicines, and triggers restocking.
Super User (Wholesaler): Receives and processes restocking orders.

The system ensures:

Real-time inventory updates
Automated low-stock alerts
Expiry monitoring
Seamless communication between seller and wholesaler
1.3 Definitions
Stock: Number of available medicine units
Reorder Level: Minimum threshold for stock
Salt: Chemical composition of medicine
Order: Request sent to wholesaler for restocking
2. Overall Description
